# 習題 1 - 請用爬山演算法解決旅行推銷員問題

這個習題我使用 **爬山演算法** 結合 **2-opt 演算法** 來求旅行推銷員問題。透過定義合適的狀態評估函數，讓演算法能在鄰域空間中不斷尋找更短的巡迴路徑，最終收斂至局部最佳解。

---

## 設計思路

在處理TSP時，我的目標是將總路徑距離最小化。但是傳統的爬山演算法通常是設計來「尋找最大值」。為了達到目的，用了以下兩個關鍵機制：

### 1.狀態評估：Height 函數
為了配合爬山演算法尋找最大值的特性，我將評估函數定義為：**Height = -totalDistance**。

這樣一來，**總距離越短，Height 值就越大**。當演算法往 Height 更高的鄰居移動時，本質上就是在縮短走完整個城市迴圈的總距離。

### 2.鄰居產生機制：2-opt 局部搜尋
為了避免隨機亂猜，我採用了經典的 **2-opt 演算法** 來產生鄰居。
* **原理**：在當前路徑中任選兩條不相交的邊 $(a, b)$ 與 $(c, d)$ 並將其打斷，然後重新連接成 $(a, d)$ 與 $(b, c)$。
* **搜尋策略**：每次迭代都會窮舉所有可能的 2-opt 交換組合，並從中挑選出一個 **Height 最高的最佳鄰居** 作為下一步的移動方向。

---

## 程式碼架構說明

程式碼主要由以下幾個核心函式組成：

* `get_distance(city1, city2)`: 呼叫 `math.dist` 計算兩個城市座標之間的歐幾里得距離。
* `height(solution, cities)`: 計算給定路徑的總長度，並回傳其負值作為爬山法的適應度指標。
* `get_best_neighbor(solution, cities)`: 透過雙層迴圈窮舉 2-opt 交換，找出當前路徑周圍所有鄰居中表現最好的那一個。
* `hill_climbing(cities)`: 演算法主迴圈。從初始路徑開始，不斷移向更好的鄰居。當周圍所有鄰居的 Height 都無法超越當前解時，代表已達局部最佳解，演算法隨即停止並輸出結果。
