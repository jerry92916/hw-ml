# 旅行推銷員問題 (TSP) - 爬山演算法與 2-opt 局部搜尋實作

這個專案使用 **爬山演算法 (Hill Climbing)** 結合 **2-opt 演算法** 來求解經典的**旅行推銷員問題 (Traveling Salesperson Problem, TSP)**。透過定義合適的狀態評估函數（Height Function），讓演算法能在鄰域空間中不斷尋找更短的巡迴路徑，最終收斂至局部最佳解（Local Optimum）。

---

## 💡 核心設計思路

在處理 TSP 時，我們的目標是「將總路徑距離最小化」。然而，傳統的爬山演算法通常是設計來「尋找最大值」（往高處爬）。為了完美結合這兩者，我實作了以下兩個關鍵機制：

### 1. 狀態評估：Height 函數
為了配合爬山演算法尋找最大值的特性，我將評估函數（Height 值）定義為：**Height = -totalDistance**。

這樣一來，**總距離越短，Height 值就越大**。當演算法往 Height 更高的鄰居移動時，本質上就是在縮短走完整個城市迴圈的總距離。

### 2. 鄰居產生機制：2-opt 局部搜尋
為了避免隨機亂猜，我採用了經典的 **2-opt 演算法** 來產生鄰居。
* **原理**：在當前路徑中任選兩條不相交的邊 $(a, b)$ 與 $(c, d)$ 並將其打斷，然後重新連接成 $(a, d)$ 與 $(b, c)$。
* **程式碼實作**：利用 Python 的切片（Slicing）語法 `[::-1]`，將指定範圍內的路徑進行**反轉**，高效地模擬出邊交換的效果。
* **搜尋策略**：每次迭代都會窮舉所有可能的 2-opt 交換組合，並從中挑選出一個 **Height 最高（亦即距離縮短最多）的最佳鄰居** 作為下一步的移動方向。

---

## 🛠️ 程式碼架構說明

程式碼主要由以下幾個核心函式組成：

* `get_distance(city1, city2)`: 呼叫 `math.dist` 計算兩個城市座標之間的歐幾里得距離（Euclidean distance）。
* `height(solution, cities)`: 計算給定路徑的總長度，並回傳其負值作為爬山法的適應度（Fitness）指標。
* `get_best_neighbor(solution, cities)`: 透過雙層迴圈窮舉 2-opt 交換，找出當前路徑周圍所有鄰居中表現最好的那一個。
* `hill_climbing(cities)`: 演算法主迴圈。從初始路徑（依城市編號順序）開始，不斷移向更好的鄰居。當周圍所有鄰居的 Height 都無法超越當前解時，代表已達局部最佳解，演算法隨即停止並輸出結果。
