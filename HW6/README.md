# 習題 6 -- 請用非 transformer (attention) 的方法，做一個語言模型

在這個習題中，我實作了一個以決策樹分類器為核心的統計語言模型。不同於傳統馬可夫鏈透過轉移機率矩陣或次數統計表來預測下一個字，本專案將文本續寫問題轉換為機器學習中的多分類問題，利用決策樹學習中文文本中的字詞規律與上下文關係，進而完成自動文字接龍與互動式文字預測。透過這次實作，我除了了解統計語言模型的基本概念外，也學習到如何將自然語言處理問題轉化為分類任務，並實際應用機器學習方法來建立文字生成模型。

---

## 核心設計與運作原理

模型核心架構是二階馬可夫鏈，意味著模型在預測下一個字時，只取決於當前的「最後兩個字」（當前狀態），與更久以前的歷史無關。
### 1. 機器學習管線
使用 `scikit-learn` 的 `Pipeline` 將特徵工程與模型訓練完美封裝：
* **CountVectorizer**：自訂 `analyzer` 以支援中文單字元的 Tokenization，將連續兩個字的狀態轉換為稀疏向量。
* **DecisionTreeClassifier**：以 `entropy`作為分割標準，將文字接龍的邏輯構建為一棵巨大的決策森林，每個葉子節點對應到原始文本中各個中文字出現的機率。

### 2. 互動模式與機率採樣
模型訓練完成後，會提供一個互動式的終端機介面：
* **Top-3 預測機率**：當你輸入一段文字，模型會精準算出最可能接在後面的前 3 個中文字及其精確百分比機率。
* **隨機文字接龍 **：不單純只選機率最高的字（避免生成死板、無限重複的句子），而是根據決策樹輸出的預測機率分布（`predict_proba`）進行**權重隨機採樣**，自動往後延伸 20 個字，讓生成的句子更具多樣性與趣味。

## 資料集與預處理說明

* **動態文本下載**：程式啟動時會自動從 GitHub 遠端下載開源中文語料文本 `tx.txt`。
* **自動容錯機制**：若遇網路中斷或下載失敗，系統會無縫切換至內建的中文測試短文本，確保程式在任何環境下皆能正常示範。
* **文本清洗**：自動移除文本內所有空白與換行符號，確保馬可夫鏈的狀態滑動視窗不會被格式符號干擾。

## 測試與互動場景

當你啟動程式並完成模型訓練後，你可以嘗試以下幾種輸入情境：

1. **常規詞彙測試**：輸入 `台灣`、`我們` 或 `發展`，模型會展現出從網頁文本中學到的政治、經濟或社會日常用語，並順暢地往下接龍。
2. **死路攔截機制**：如果你輸入了訓練文本中「從未出現過」的兩字組合，程式內建的異常處理（`try-except`）會溫和地提醒你該組合查無資料，並提供推薦詞彙，避免程式崩潰。
