# 習題 5 -- 請把 agent0.py 擴充，讓他有安全控管的功能


利用Gemini協助debug。  
這個習題中，我實作一個 **SecureAgent** 機制，將安全防護思維深度整合進 AI Agent 的生命週期中。透過輸入防禦、輸出遮罩、結構化輸出驗證、以及高風險操作的人工介入，建立起一套多層次的 AI 安全護欄，有效防止提示詞注入與敏感隱私資料的外洩。

---

## 核心安全防護機制 

我將 AI 互動的完整生命週期切分為四道安全防線，確保任何階段的潛在威脅都能被精準攔截：
### 1. 輸入端安全控管：在使用者的請求送入大語言模型之前，系統會先利用正則表達式進行檢查與過濾，攔截常見的惡意攻擊樣式，例如提示詞注入與系統命令注入。透過事前防禦的方式，避免使用者利用惡意指令改變模型原有行為，降低系統遭受攻擊的風險。


### 2. 輸出端結構化驗證：系統採用 Pydantic 定義固定的回應格式，並搭配 OpenAI 的結構化解析機制，確保模型輸出的內容符合預先設定的資料結構。即使模型受到干擾，也無法以非結構化文字繞過系統邏輯，若輸出格式不符規範，系統也能及時捕捉錯誤並停止後續處理，以提升回應的穩定性與可靠性。


### 3. 工具執行安全控管：當 Agent 判斷需要呼叫外部工具時，系統會依據工具的風險程度進行分級管理。低風險操作可自動執行並記錄日誌，而涉及刪除資料、執行系統指令等高風險操作時，則必須經過管理員手動輸入授權後才能執行，避免 AI 在無人監督的情況下做出具有破壞性的行為。


### 4. 敏感資料過濾與遮罩：在回覆內容提供給使用者之前，系統會再次進行安全審查，自動掃描電子郵件、API 金鑰等敏感資訊，並以 [REDACTED] 方式進行遮罩處理。藉由最後一道檢查機制，可有效降低模型在推理過程中意外洩漏機密資料的風險，進一步保護使用者與系統的隱私安全。

## 模組超參數與環境配置

* **基礎核心**：基於 `gpt-4o` 模型進行高精確度的意圖解析與結構化輸出。
* **安全日誌**：內建 `logging` 追蹤機制，所有安全警報皆會同步輸出至控制台，並永久保存至本地的 `agent_security.log` 檔案中，便於後續的資安審計。

## 驗證與測試場景

內建三大完整測試情境，用以驗證安全防禦的有效性：

1. **正常安全請求測試**：驗證常規對話與流程順暢度。
2. **提示詞注入攻擊測試**：輸入惡意的逆向工程指令，驗證系統是否能成功熔斷並給出「系統檢測到不安全的請求，拒絕處理」的安全回覆。
3. **隱私資料外洩防禦測試**：模擬 LLM 意外吐出敏感金鑰時，驗證系統能否在毫秒級內完成 `[REDACTED]` 遮罩。
